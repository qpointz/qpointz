## Scope

### URN grammar (SPEC §5.2)

Universal Mill URN format:
```
urn:mill/<group>/<class>:<id>
```

Confirm and document:

| URN | Group | Class | Owner |
|-----|-------|-------|-------|
| `urn:mill/metadata/facet-type:<name>` | `metadata` | `facet-type` | metadata-core |
| `urn:mill/metadata/scope:<name>` | `metadata` | `scope` | metadata-core |
| `urn:mill/model/schema:<id>` | `model` | `schema` | `mill-data-schema-*` |
| `urn:mill/model/table:<schema>.<table>` | `model` | `table` | `mill-data-schema-*` |
| `urn:mill/model/attribute:<schema>.<table>.<col>` | `model` | `attribute` | `mill-data-schema-*` |

Entity URNs in `metadata_entity.entity_res` are not restricted to any group — the metadata
service stores whatever valid `urn:mill/` URN the caller provides.

**Case-insensitivity rule:** URNs are case-insensitive; canonical form is lowercase. Repositories
MUST normalise to lowercase before persistence and comparison. `MetadataEntityUrn.canonicalize(urn)`
is the standard normaliser — called at service boundaries before any repository call.

### `FacetInstance` type contract (SPEC §5.4)

```kotlin
data class FacetInstance(
    val uid: String,            // stable UUID, assigned at creation, immutable
    val entityId: String,       // entity URN
    val facetTypeKey: String,   // facet type URN
    val scopeKey: String,       // scope URN
    val mergeAction: MergeAction, // SET | TOMBSTONE | CLEAR
    val payload: Map<String, Any?>,
    val createdAt: Instant,
    val createdBy: String?,
    val lastModifiedAt: Instant,
    val lastModifiedBy: String?
)
```

`enum class MergeAction { SET, TOMBSTONE, CLEAR }` — semantics in SPEC §8.3.

`SINGLE` cardinality: at most one row per `(entityId, facetTypeKey, scopeKey)`.
`MULTIPLE` cardinality: multiple rows; each row identified by its unique `uid`.

### Unified operation audit table (SPEC §8.4) — `metadata_audit`

**Normative name:** `metadata_audit` (replaces legacy `metadata_operation_audit` and draft `metadata_facet_investigation_audit`).

```sql
CREATE TABLE metadata_audit (
    audit_id         BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    operation        VARCHAR(64)  NOT NULL,
    subject_type     VARCHAR(32)  NOT NULL,
    subject_ref      VARCHAR(512),
    actor_id         VARCHAR(255),
    correlation_id   VARCHAR(255),
    occurred_at      TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP,
    payload_before   TEXT,
    payload_after    TEXT
);
```

**No** row-audit quad on `metadata_audit`. **No** FKs to facet/entity tables. **Fail-open** listeners per SPEC.

**Audit ownership:** **Only** JPA entity listeners invoke `MetadataAuditRepository`. **No** service or controller dependency on audit — SPEC §6.5 / §8.4.

### REST surface (SPEC §10)

Locked endpoint list — all changes from old API enumerated:

| Change | Detail |
|--------|--------|
| `GET /entities` | `?schema=`/`?table=` params removed; `?kind=` added (opaque) |
| Entity `{id}` path | Dot-path ids return 400 |
| `MetadataEntityDto` | No coordinate fields, no `facets` field, no `type` enum |
| `DELETE .../facet-instances/{uid}` | Endpoint removed; use **`DELETE .../facets/{typeKey}/{facetUid}`** (uid **in path**, SPEC §10.2) |
| `MULTIPLE` delete | Target the specific assignment by **`facetUid`** in the path; use `DELETE .../facets/{typeKey}?scope=...` only for **unassignAll** at a scope |
| **Unassign semantics** | **§10.2** — physical **DELETE** only for **`merge_action == SET`** in **non-overlay** scopes (v1: **GLOBAL** + whitelist); else **TOMBSTONE** |
| **Merge trace (UI)** | **`GET .../entities/{id}/facets/merge-trace?context=...`** — exposes per-scope contributions including **`mergeAction`**; **`FacetInstanceDto`** / **`GET .../facets`** omit **`mergeAction`** |

### Platform facet type seeds (SPEC §8.5)

Owned by metadata migration (greenfield SQL, e.g. `V4__metadata_greenfield.sql`):
- `urn:mill/metadata/facet-type:descriptive`
- `urn:mill/metadata/facet-type:concept`

**Source of truth:** `platform-facet-types.json` for metadata-owned types; **one-time** SQL `INSERT`s in **`V4__metadata_greenfield.sql`**, then **manual** JSON/SQL sync.

Registered at startup by `SchemaFacetAutoConfiguration` (NOT in the squashed metadata migration):
- `urn:mill/metadata/facet-type:structural`
- `urn:mill/metadata/facet-type:relation`
- `urn:mill/metadata/facet-type:value-mapping`
