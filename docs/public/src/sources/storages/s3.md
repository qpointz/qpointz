# AWS S3

The `s3` storage type reads files from an Amazon S3 bucket. Readers, table mapping, and all other source configuration options work exactly the same as with [local storage](local.md) — only the `storage` block changes.

---

## Configuration

```yaml
storage:
  type: s3
  bucket: my-data-bucket
  prefix: warehouse/parquet/
  region: eu-west-1
```

| Property   | Required | Default | Description |
|------------|----------|---------|-------------|
| `bucket`   | yes      | —       | S3 bucket name. |
| `prefix`   | no       | —       | Key prefix to limit the scan scope (acts like a virtual directory). |
| `region`   | no       | —       | AWS region. Falls back to SDK default region resolution when omitted. |
| `endpoint` | no       | —       | Custom S3 endpoint URL. Use for S3-compatible services or emulators (e.g. MinIO). |
| `requesterPays` | no  | `false` | Set to `true` for [Requester Pays](https://docs.aws.amazon.com/AmazonS3/latest/userguide/RequesterPaysBuckets.html) buckets. |

---

## Authentication

When `storage.auth` is omitted, Mill uses the **AWS default credential chain** (environment variables, instance profile, SSO, etc.). This is the recommended approach for production.

For explicit credentials, provide `accessKey` and `secretKey`:

```yaml
storage:
  type: s3
  bucket: my-data-bucket
  auth:
    accessKey: ${AWS_ACCESS_KEY_ID}
    secretKey: ${AWS_SECRET_ACCESS_KEY}
```

| Auth property        | Description |
|----------------------|-------------|
| `accessKey`          | AWS access key ID. |
| `secretKey`          | AWS secret access key. |
| `sessionToken`       | Optional session token for temporary credentials. |
| `preferAmbientCredentials` | When `true`, forces ambient credential resolution even if explicit keys are present. |

Do not commit raw credentials to version control. Use environment variable expansion (`${VAR}`) or a secret store.

## IAM Permissions

Mill requires **read-only** access to list and retrieve objects. The minimal IAM policy:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "ListBlobs",
      "Effect": "Allow",
      "Action": "s3:ListBucket",
      "Resource": "arn:aws:s3:::YOUR-BUCKET-NAME"
    },
    {
      "Sid": "ReadObjects",
      "Effect": "Allow",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::YOUR-BUCKET-NAME/*"
    }
  ]
}
```

| Action | ARN pattern | Purpose |
|--------|-------------|---------|
| `s3:ListBucket` | `arn:aws:s3:::bucket` | List objects under prefix |
| `s3:GetObject` | `arn:aws:s3:::bucket/*` | Read object content and metadata (`HeadObject` uses the same permission) |

!!! warning "Two ARN patterns"
    S3 permissions require **two** resource ARNs:
    
    - **Bucket-level** (`arn:aws:s3:::bucket`) — for `ListBucket`
    - **Object-level** (`arn:aws:s3:::bucket/*`) — for `GetObject`
    
    A policy with only the bucket ARN will allow listing but fail on reads with `AccessDeniedException`.

For least-privilege, scope the object ARN to match your prefix:

```json
"Resource": "arn:aws:s3:::analytics-data/warehouse/parquet/*"
```

### Requester Pays buckets

If the bucket has [Requester Pays](https://docs.aws.amazon.com/AmazonS3/latest/userguide/RequesterPaysBuckets.html) enabled and your IAM principal is not the bucket owner, set `requesterPays: true`:

```yaml
storage:
  type: s3
  bucket: public-datasets
  prefix: opendata/
  requesterPays: true
```

Without this flag, S3 returns **403 Forbidden** on read operations even when IAM permissions are correct.

## Local Development with MinIO

For local development and CI, use [MinIO](https://min.io/) as an S3-compatible emulator:

```yaml
storage:
  type: s3
  bucket: test-bucket
  region: us-east-1
  endpoint: http://localhost:9000
  auth:
    accessKey: minioadmin
    secretKey: minioadmin
```

## Example

Complete source descriptor reading Parquet files from S3:

```yaml
name: warehouse
storage:
  type: s3
  bucket: analytics-data
  prefix: warehouse/
  region: us-east-1
readers:
  - type: parquet
    table:
      mapping:
        type: directory
        depth: 1
```
