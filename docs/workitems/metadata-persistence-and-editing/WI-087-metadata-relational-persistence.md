# WI-087 — Metadata Relational Persistence

Status: `planned`
Type: `feature`
Area: `metadata`
Backlog refs: `M-11`, `M-12`

## Goal

Create a new `metadata/mill-metadata-persistence` module with JPA entities, Spring Data
repositories, and adapter implementations of `MetadataRepository` and `FacetTypeRepository`.
Add Flyway `V4__metadata.sql` seeding five platform facet types. Wire via
`MetadataJpaPersistenceAutoConfiguration` behind `mill.metadata.storage.type=jpa`.

## Dependencies

- WI-086 must have deleted `MetadataTargetType` enum and changed `applicableTo` to `Set<String>`
  before this WI ships (seed SQL uses plain string arrays).
- Can run in parallel with WI-086 on a separate branch segment and merged together.

## Mill URN Impact

Facet type keys (`type_key`) and scope keys in the DB use the Mill URN scheme defined in WI-086.
`MetadataUrns` (from `mill-metadata-core`) provides all constants.

Seed `type_key` values for platform facet types:
- `urn:mill/metadata/facet-type:descriptive`
- `urn:mill/metadata/facet-type:structural`
- `urn:mill/metadata/facet-type:relation`
- `urn:mill/metadata/facet-type:concept`
- `urn:mill/metadata/facet-type:value-mapping`

Seed `applicable_to_json` values use entity-type URN strings, not enum names:
- `["urn:mill/metadata/entity-type:schema","urn:mill/metadata/entity-type:table","urn:mill/metadata/entity-type:attribute"]`

Scope key `scope_id` for the global scope: `urn:mill/metadata/scope:global`.

The `JpaMetadataRepository.resolveOrCreateScope(scopeKey)` and
`JpaMetadataRepository.toScopeKey(entity)` methods use `MetadataUrns` to encode/decode scope keys.

## Adapters — `io.qpointz.mill.persistence.metadata.jpa.adapters`

### `JpaMetadataRepository` implements `MetadataRepository`

- `toDomain(record, facetRows, scopeEntities)` — for each `MetadataFacetScopeEntity`: resolve
  `scopeId → scopeKey` string via `toScopeKey(MetadataScopeEntity)`, deserialise
  `payloadJson → Map<String,Any?>` via Jackson. Reconstruct `MetadataEntity.facets`.
- `save(entity)` `@Transactional` — upsert `metadata_entity`, then for each
  `(facetType, scopeMap)` entry: `resolveOrCreateScope(scopeKey) → scopeId`, upsert
  `metadata_facet_scope`.
- `resolveOrCreateScope(scopeKey: String): String` — parse scope key, query
  `MetadataScopeJpaRepository.findByScopeTypeAndReferenceId`; create row if absent.
- `toScopeKey(scope: MetadataScopeEntity): String` — encodes `"global"`, `"user:<ref>"`, etc.

### `JpaFacetTypeRepository` implements `FacetTypeRepository`

- `toDomain(entity)` — parse `applicableToJson → Set<String>`, `contentSchemaJson → Map?`.
- `toEntity(descriptor)` — serialise both fields to JSON strings via Jackson `ObjectMapper`.

### `JpaMetadataChangeObserver` implements `MetadataChangeObserverDelegate`

- `onEvent(event: MetadataChangeEvent)` — implements the `MetadataChangeObserver` fun interface
  from `mill-metadata-core`. Converts `MetadataChangeEvent` to `MetadataOperationAuditEntity`:
  maps `event.entityId`, `event.actorId`, `event.occurredAt`, and the sealed-class subtype to
  `operationType`; serialises `payloadBefore`/`payloadAfter` (from `EntityUpdated`, `FacetUpdated`,
  etc.) to JSON strings stored in `@Lob` CLOB columns via Jackson.
- `@Async` — fire-and-forget. Exceptions are caught and logged at `WARN`, never rethrown.
  Audit failures must not block or roll back the triggering operation.
- Log at `INFO` level for every audit entry persisted (operation type, entity id, actor).

## Autoconfiguration

`MetadataJpaPersistenceAutoConfiguration` in `mill-metadata-autoconfigure`:

```kotlin
@AutoConfiguration
@ConditionalOnClass(name = ["io.qpointz.mill.persistence.metadata.jpa.entities.MetadataEntityRecord"])
@ConditionalOnProperty(prefix = "mill.metadata.storage", name = ["type"], havingValue = "jpa")
@EntityScan(basePackages = ["io.qpointz.mill.persistence.metadata.jpa.entities"])
@EnableJpaRepositories(basePackages = ["io.qpointz.mill.persistence.metadata.jpa.repositories"])
class MetadataJpaPersistenceAutoConfiguration {
    @Bean @ConditionalOnMissingBean(MetadataRepository::class)
    fun jpaMetadataRepository(...): MetadataRepository = JpaMetadataRepository(...)

    @Bean @Primary @ConditionalOnMissingBean(FacetTypeRepository::class)
    fun jpaFacetTypeRepository(...): FacetTypeRepository = JpaFacetTypeRepository(...)

    @Bean  // plain @Bean — the chain collects all MetadataChangeObserverDelegate beans
    fun jpaMetadataChangeObserver(...): MetadataChangeObserverDelegate = JpaMetadataChangeObserver(...)
    // JpaMetadataScopeRepository added in WI-089; JpaMetadataPromotionRepository deferred to WI-091
}
```

- `apps/mill-service/build.gradle.kts` — add `runtimeOnly(project(":metadata:mill-metadata-persistence"))`

## Acceptance Criteria

- `mill.metadata.storage.type=jpa` activates JPA repository; `file` (default) keeps file-backed.
- All five platform facet types present in DB after Flyway V4 runs.
- `applicableTo` stored and retrieved as `Set<String>` with no enum references.
- `save/findById` round-trip preserves all facets across all scopes.
- Every write operation records an entry in `metadata_operation_audit` with JSON CLOBs.
- Audit failure does not block or roll back the triggering write operation.
- Integration tests green against H2.
- `:metadata:mill-metadata-persistence:test` and `:testIT` both green.
