## Goal

Deterministic conversation replay without OpenAI: scripted model responses drive real [`LangChain4jAgent`](../../../../ai/mill-ai/src/main/kotlin/io/qpointz/mill/ai/runtime/langchain4j/LangChain4jAgent.kt) against a selected `profileId`.

## LangChain4j agent loop (what script must satisfy)

From [`LangChain4jAgent.kt`](../../../../ai/mill-ai/src/main/kotlin/io/qpointz/mill/ai/runtime/langchain4j/LangChain4jAgent.kt) (`MAX_ITERATIONS`, lines ~166–232):

1. **Model call** → if no tool requests → `AnswerCompleted` → turn ends.
2. **Model call** → tool requests → for each request:
   - Emit `ToolCall(name, args, iteration)`
   - Run real handler → emit `ToolResult(name, content)`
   - Append `ToolExecutionResultMessage.from(toolRequest, resultText)` to message list (LangChain4j pairs result to request **by tool request id** generated by the model stub)
3. **CAPTURE path** (successful capture tool): optional **`LangChain4jProtocolExecutor`** → **additional model call** → `ProtocolFinal` → `AnswerCompleted("")` → turn ends (**no further planner steps**).
4. **QUERY path** (e.g. `validate_sql`): loop continues → **another model call** unless coordinator ends turn (WI-304: coordinator emits `ProtocolFinal` then agent still needs final `answer` script step).

## Script semantics (`ask.script[]`)

Each entry is **one model invocation** consumed in order from a queue shared by:

- **Planner loop** (`LangChain4jAgent.complete`)
- **Protocol executor** (`LangChain4jProtocolExecutor` on CAPTURE / future executor paths)

### Entry shapes

| Entry | Maps to | Notes |
|-------|---------|-------|
| `toolCalls:` | `AiMessage` with `ToolExecutionRequest`(s) | Harness assigns synthetic `id` per request (e.g. `call-0`, `call-1`) if omitted in YAML; must match `ToolExecutionResultMessage.from(request, …)` |
| `answer:` | Plain `AiMessage` text | Final planner response or protocol executor JSON (STRUCTURED_FINAL) |

### Multi-step QUERY turn (SQL emit — **two script steps**)

```yaml
script:
  - toolCalls:
      - name: validate_sql
        args: { sql: "SELECT 1", attempt: 1 }
  - answer: ""
```

Runtime sequence (not scripted): `ToolCall` → `ToolResult` → coordinator `ProtocolFinal` (WI-304, no executor model) → second model step returns empty answer → `AnswerCompleted`.

### CAPTURE turn (facet — **two script steps minimum**)

```yaml
script:
  - toolCalls:
      - name: propose_facet_assignment
        args: { ... }
  - answer: '{"facetTypeKey":"descriptive",...}'   # protocol executor structured JSON
```

Protocol executor consumes the **second** script entry as its model response.

### Script exhaustion

If the agent or protocol executor needs another model call but the script queue is empty → **fail fast** with message naming: profile, turn index, last event, script index expected.

### Tool call validation (optional, recommended)

YAML may declare `expectTools: [validate_sql]` on script steps; runner asserts actual tool names match before executing handlers.
