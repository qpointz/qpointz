# WI-040 — AI v3 MCP Capability Exposure Plan

Status: `planned`  
Type: `📝 docs`  
Area: `ai`, `platform`  
Backlog refs: `TBD`

## Problem Statement

Capabilities are expected to be reusable outside Mill in the future, but the path to MCP exposure is not yet defined.

## Goal

Define how `v3` capabilities can later be exposed through MCP without redesigning the capability model.

## In Scope

1. Map capability assets to MCP concepts.
2. Clarify in-process versus external usage expectations.
3. Identify constraints that `v3` must avoid from the start.

## Out of Scope

- Implementing the MCP server.

## Acceptance Criteria

- A clear future MCP exposure path exists for `v3` capabilities.
- The capability model avoids in-process-only assumptions that would block MCP later.

## Deliverables

- This work item definition (`docs/workitems/WI-040-ai-v3-mcp-capability-exposure-plan.md`).
