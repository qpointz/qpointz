# WI-037 — AI v3 Core Domain Vocabulary

Status: `planned`  
Type: `📝 docs`  
Area: `ai`  
Backlog refs: `TBD`

## Problem Statement

The `v3` design references core concepts such as capability, profile, protocol, run state, and artifact, but they need a stable shared vocabulary.

## Goal

Define the core domain terminology used across the `v3` runtime.

## In Scope

1. Define the meanings of capability, descriptor, provider, agent profile, context, protocol, artifact, planner step, and run state.
2. Keep terminology aligned with the agentic design documents.

## Out of Scope

- Concrete implementation classes.

## Acceptance Criteria

- `v3` core concepts are named and described consistently.
- The vocabulary is sufficient for capability, runtime, and MCP-facing design work.

- This work item definition (`docs/workitems/WI-037-ai-v3-core-domain-vocabulary.md`).
