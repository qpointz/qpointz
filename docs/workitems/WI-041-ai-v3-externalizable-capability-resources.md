# WI-041 — AI v3 Externalizable Capability Resources

Status: `planned`  
Type: `📝 docs`  
Area: `ai`, `platform`  
Backlog refs: `TBD`

## Problem Statement

MCP-friendly design requires capability assets beyond tools to be treated as externalizable resources.

## Goal

Identify which capability assets should be modeled as first-class externally exposable resources.

## In Scope

1. Identify externalizable assets such as prompts, protocols, descriptors, examples, and artifact schemas.
2. Clarify which assets are better modeled as tools, prompts, or resources.

## Out of Scope

- Resource transport implementation.

## Acceptance Criteria

- `v3` capability design is not limited to tools-only exposure.
- The resource categories needed for future MCP exposure are identified.

- This work item definition (`docs/workitems/WI-041-ai-v3-externalizable-capability-resources.md`).
