# WI-038 — AI v3 Capability Model and Descriptor Format

Status: `planned`  
Type: `✨ feature`  
Area: `ai`  
Backlog refs: `TBD`

## Problem Statement

`v3` depends on capabilities as its primary building block, but the capability contract and descriptor format are not yet fixed.

## Goal

Define the `v3` capability model and descriptor format.

## In Scope

1. Describe capability contents: prompts, tools, protocols, policies, reasoning descriptors, and tags.
2. Define capability descriptor fields.
3. Ensure the design remains compatible with future MCP exposure.

## Out of Scope

- Runtime discovery implementation.

## Acceptance Criteria

- A concrete capability contract is defined.
- Capability descriptors are self-describing and externally usable.

- This work item definition (`docs/workitems/WI-038-ai-v3-capability-model-and-descriptor-format.md`).
