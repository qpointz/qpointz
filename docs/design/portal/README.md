# Portal

Design notes for a federated portal that sits above individual Mill instances and
provides a company-wide view of the data landscape.

## Classification Criteria

A document belongs here if its primary subject is one of:

- Multi-instance federation and central landscape views
- Cross-instance metadata aggregation and normalization
- Portal-specific collaboration workflows between consumers and maintainers
- Central registry of known facet types and rendering/validation rules
- Metadata scoring, stewardship, and company-level knowledge sharing

## Does NOT Belong Here

- Single-instance metadata service internals -> `metadata/`
- Mill runtime/service implementation details -> `platform/`
- General UI patterns not specific to the portal domain -> `ui/`
